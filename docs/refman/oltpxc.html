NAME

oltpxc − RWP*Load Simulator oltpxc table

DECLARATION

When using external control of an oltprun, the oltpxc table is used to set values that will be picked up by the current run. The primary key of the table is the key and the hostname, and the remaining columns are set via calls to oltpxcset. Alternatively, you can arrange for some other program set set values.

create table oltpxc
( key varchar2(30) not null
, hostname varchar2(64) not null
, ratefactor number
, loadfactor number
, hardparse number
, parsefailure number
, ignore_parsefailure number
, minpool number
, maxpool number
, cursorleak number
, sessionleak number
, cursorcache number
, logoffrate number
, badplan number
, stopnow number
, cpusec number
, waitsec number
, constraint oltpxc_pk primary key(key, hostname)
)

COLUMNS

key

This is the key provided to oltprun or oltpcore, with the default value taken from the RWLOLTP_NAME environment variable.

hostname

The hostname of the system where oltprun or oltpcore is running.

ratefactor

A factor that is multiplied to the standard ratefactor, i.e. the transaction arrival rate.

loadfactor

The value is the proportion of worker threads that are active.

hardparse

Sets the probability that certain sqlstatements use literals in stead of bind variables, and therefore causing hard parsing to happen.

parsefailure

Sets the probability that certain sqlstatements use either an incorrect table name or some syntax error during parsing causing respectively ORA-00942, ORA-00933 or ORA-00936.

ignore_parsefailure

If set to non-zero before starting oltpxcrun, parse failures from the previous setting will not be printed to stderr as they normally do.

logoffrate

The proportion of sessions that, when returned to the pool, will also do a disconnect from the database.

minpool, maxpool

Set the minimum and maximum size of the session pool in each process.

cursorleak

Set the proportion of cursors that will be leaked.

sessionleak

Set the proportion of sessions that will be leaked. Note that after a session leak, you cannot recover the current run, so it will be need to be aborted using stopnow.

cursorcache

Set the size of the cursor cache.

badplan

Set to the number of queries (0..10) that will do full table scan rather than index lookup.

cpusec

If positive, there will be a cpu busy loop lasting cpusec seconds in all frequently executed transactions. Typical values would be up to around 0.01, i.e. 10ms.

waitsec

If positive, there will be a wait lasting waitsec seconds in all frequently executed transactions. Typical values would be up to around 0.01, i.e. 10ms.

stopnow

If set to 1, the run will abort, and if you are using oltpxcrun, the next run will start. If set to 2, the run will abort, and if you are using oltpxcrun, it will also terminate. If set to 3 and you are using oltpxcrun, it will terminate when the current run has completed normally.

NOTES

The actual order of the columns in the table may be different from what is shown here, you should therefore always explicitly name columns in your sql statements.
