NAME

bengsig − The RWP*Load Simulator primary developer Bjørn Kisbye Engsig.·.

BIOLOGY

Bjørn Kisbye Engsig was born in 1958, graduated as an engineer in 1983 and has been working with the Oracle database since 1985. In a total of more than twenty-five years, he has been an employee of Oracle Corporation.

OLTP

Measuring performance and scalability of OLTP workloads has always been a challenge between simplicity and reality. Mr. Engsig has been working with and developing several tools for simulating OLTP database load, and started working on the RWP*Load Simulator in 2017. Lots of experience from actual customer workloads and from other ways to investigate in particular OLTP performance has been a primary driving motivator for rwloadsim. The difficulty of using SQL*Plus for this purpose combined with the relatively large time required to use programming languages like Java or C have also been taken into account.

In 2020, it was decided to make rwloadsim publicly available as an addition to Oracle’s already large set of open source initiatives.

It is the hope that rwloadsim will find its use in many areas of simulation and measuring of database performance. Additionally, it can also prove very useful in the more general area of "scripting" as it fills out the apparent void between standard UNIX tools such as the shell, awk, SQL*Plus, and Oracle Call Interface.
